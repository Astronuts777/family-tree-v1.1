# Family Tree v1.1

一个用于录入家庭成员信息、自动生成家谱树状图、并将数据保存到 SQLite 数据库的本地网页应用。

## 当前已完成

- 输入用户基础信息
- 输入配偶信息
- 建立家庭成员之间的关系
- 自动生成树状图
- 将成员信息保存到 SQLite 数据库

## 后续可继续扩展

- 允许其他设备访问同一份家谱数据
- 部署到云端服务器
- 接入真正的多用户共享登录系统

## 功能概览

- 录入家庭成员基础信息
  - 姓名
  - 性别
  - 出生年月日
  - 去世年月日（选填）
  - 家庭地址（选填）
  - 民族（选填）
  - 职务（选填）
  - 照片（选填）
- 录入配偶信息
  - 配偶姓名
  - 配偶出生年月日
  - 配偶去世年月日（选填）
  - 配偶职务
  - 婚姻状态
- 自动计算年龄
  - 健在成员按当前日期计算年龄
  - 已故成员按出生和去世日期计算享年
- 支持家谱关系建模
  - 父亲
  - 母亲
  - 儿子
  - 女儿
  - 兄弟
  - 姐妹
  - 女婿
  - 儿媳
  - 配偶
- 自动生成树状图
- 支持树状图缩放
- 支持成员搜索
- 支持编辑和删除成员
- 支持导出树状图为 PDF
- 使用 SQLite 持久化保存成员信息

## 项目结构

- `index.html`: 页面结构
- `styles.css`: 页面样式
- `app.js`: 前端逻辑
- `server.js`: 本地 Node + SQLite 服务
- `package.json`: 启动脚本
- `data/family-tree.db`: SQLite 数据库文件，运行后自动创建，不提交到 Git

## 运行方法

### 1. 进入项目目录

```bash
cd /Users/brittonsun/Desktop/familytree-framework
```

### 2. 启动本地服务

```bash
npm start
```

启动后会看到类似输出：

```bash
Family tree app running at http://127.0.0.1:3000
SQLite database: /Users/brittonsun/Desktop/familytree-framework/data/family-tree.db
```

### 3. 在浏览器打开页面

打开：

```text
http://127.0.0.1:3000
```

注意：

- 不要直接双击 `index.html`
- 不要使用 `file:///.../index.html` 方式打开
- 必须通过 `npm start` 启动本地服务后访问 `127.0.0.1:3000`

否则前端无法调用后端接口，SQLite 数据库不会更新。

## 如何查看数据库

### 在 IntelliJ 中查看

1. 打开 `Database` 工具窗口
2. 点击 `+`
3. 选择 `Data Source` -> `SQLite`
4. 选择数据库文件：

```text
/Users/brittonsun/Desktop/familytree-framework/data/family-tree.db
```

5. 打开 `members` 表即可查看成员数据

## 使用方法

### 新增成员

1. 输入成员基础信息
2. 如需要，输入配偶信息
3. 从第二位成员开始，选择“与谁建立关系”和“关系类型”
4. 点击“保存并加入树状图”

### 编辑成员

1. 在右侧“成员管理”里找到目标成员
2. 点击“编辑”
3. 修改表单内容
4. 再次点击“保存并加入树状图”

### 删除成员

1. 在右侧“成员管理”里找到目标成员
2. 点击“删除”

### 搜索成员

1. 在搜索框输入姓名
2. 点击“搜索”

### 导出 PDF

1. 点击“导出 PDF”
2. 在浏览器打印窗口中选择“保存为 PDF”
